คู่มือนี้พาคุณจาก "ยังไม่มีอะไรเลย" → "มีคลังความรู้การนอนของตัวเอง" ทีละขั้น

## 0. เตรียมของ 3 อย่าง (ครั้งเดียว)

| โปรแกรม | ใช้ทำอะไร | จำเป็นไหม |
|---------|-----------|-----------|
| **Claude Code** (เวอร์ชัน App หรือ VS Code) | ตัวหลัก — คุยกับ Claude สั่งงาน | ✅ ต้องมี + ล็อกอินให้เรียบร้อย |

## 1. โหลดเทมเพลตนี้ (กดปุ่มเดียว — ไม่ต้องใช้ git)

1. ไปหน้า repo นี้บน GitHub
2. กดปุ่มเขียว **`< > Code`** → เลือก **Download ZIP**
3. เปิดไฟล์ ZIP ที่โหลดมา (ดับเบิลคลิก) → ได้โฟลเดอร์ชื่อ `whoop-lab-by-golf` (หรือคล้ายๆ)
4. **ย้ายโฟลเดอร์นั้นไปไว้ที่ที่หาง่าย** เช่น Desktop หรือ Documents

> เท่านี้ก็ได้ไฟล์ครบแล้ว ไม่ต้องรู้จัก git เลย 🎉

---

## 2. เปิดโฟลเดอร์ใน Claude Code

**ถ้าใช้ Claude Code แบบ Desktop app:**
- เปิดแอป → **Open Folder / Open Project** → เลือกโฟลเดอร์ที่เพิ่งแตกไฟล์มา

**ถ้าใช้ VS Code (มี Claude Code extension):**
- VS Code → **File → Open Folder** → เลือกโฟลเดอร์นั้น → เปิดแผง Claude Code

พอเปิดถูกโฟลเดอร์ Claude จะอ่าน `CLAUDE.md` อัตโนมัติ และรู้ตัวว่าเป็น **"ผู้เชี่ยวชาญเรื่องการนอน"** ทันที
ลองพิมพ์ `/` จะเห็นคำสั่ง **`/ingest` `/wiki-link` `/advise`** โผล่มา

> ⚠️ **สำคัญ:** คำสั่งพวกนี้จะติดก็ต่อเมื่อเปิด "โฟลเดอร์เทมเพลตนี้" เป็นโปรเจค
> ถ้าเปิดโฟลเดอร์อื่นอยู่จะไม่เห็น `/ingest` — ให้เปิดโฟลเดอร์นี้ก่อนเสมอ

---

## 3. เอาความรู้เข้าคลัง (ครั้งแรก)

1. เปิด https://notebooklm.google.com → **New notebook** → **Add source → YouTube** → วาง link คลิป
   การนอนที่สนใจ (แนะนำ: Matthew Walker, Andrew Huberman)
2. เปิด transcript ที่ NotebookLM สร้าง → กด **Ctrl/Cmd + A** เลือกทั้งหมด → **copy**
3. กลับมาที่ Claude Code → **วาง transcript** ลงไป แล้วพิมพ์:
   ```
   /ingest
   ```
4. Claude จะ clean + สรุปเป็นภาษาไทย + สร้างโน้ต atomic ใน `vault/` พร้อม `[[ลิงก์]]` ให้เอง
   (อาจถามชื่อคลิป / ผู้พูด / URL เพิ่ม — ตอบไปได้เลย)

> 💡 **วาง transcript ดิบยาวๆ ตรงๆ อย่าสรุปเองก่อน** — Claude ต้องเห็นตัวเลขและรายละเอียดครบ
> เพื่อกลั่นโน้ตให้แม่น (ถ้าปุ่ม `/ingest` ยังไม่โผล่ พิมพ์ภาษาไทยธรรมดาว่า
> "ช่วยสรุป transcript นี้ลง vault ตามกติกาใน CLAUDE.md" ก็ได้ผลเหมือนกัน)

ทำซ้ำข้อ 3 กับหลายๆ คลิป คลังจะโตขึ้นเรื่อยๆ

---

## 4. เชื่อมโยงความรู้

พอมีโน้ตหลายใบ พิมพ์:
```
/wiki-link
```
Claude จะหาความเชื่อมโยงระหว่างโน้ต เติม `[[ลิงก์]]` ที่ขาด และอัปเดตหน้าสารบัญ `MOC-Sleep.md`

---

## 5. ขอคำแนะนำ

ถามอะไรก็ได้เรื่องการนอน:
```
/advise ตอนเช้าควรโดนแดดกี่นาที ทำไม
```
Claude จะตอบโดย **อ้างอิงโน้ตในคลัง** (ไม่เดา) ถ้าคลังยังไม่มีข้อมูล มันจะบอกตรงๆ

---

## 6. (แนะนำ) เปิดดูใน Obsidian

1. เปิด Obsidian → **Open folder as vault** → เลือกโฟลเดอร์ **`vault`** (โฟลเดอร์ข้างในนะ ไม่ใช่โฟลเดอร์นอกสุด)
2. กด **Graph View** → เห็นใยความรู้เชื่อมกัน
3. เริ่มอ่านจาก `00-Index/MOC-Sleep.md`

## คำสั่งสรุป

|-------|---------|
| `/ingest` | เปลี่ยน transcript → โน้ตในคลัง |

## เจอปัญหา?
- **ไม่เห็น slash command** → เช็คว่าเปิด **โฟลเดอร์เทมเพลตนี้** เป็นโปรเจค (โฟลเดอร์ที่มี `CLAUDE.md`)
- **ยังไม่ได้ล็อกอิน Claude** → ต้องติดตั้ง + ล็อกอิน Claude Code ให้เรียบร้อยก่อนถึงจะสั่งงานได้
- **NotebookLM import คลิปไม่ได้** → คลิปต้องมี caption และไม่ใช่คลิปที่เพิ่งลง <72 ชม.
- **โน้ตออกมาเป็นภาษาอังกฤษ** → บอก Claude ว่า "ตอบไทย" ได้เลย หรือเช็คกฎใน `CLAUDE.md`

## 🧑‍💻 (สำหรับสาย dev) อยากใช้ git แทน?
ถ้าคุ้นกับ git อยู่แล้ว จะ clone หรือกด **"Use this template"** เพื่อได้ repo เป็นของตัวเองก็ได้:
```bash
git clone <URL-ของ-repo-นี้> sleep-expert
cd sleep-expert && claude
```
> แนะนำ **"Use this template"** มากกว่า clone ถ้าอยากได้ repo ของตัวเองไว้ backup โน้ต
