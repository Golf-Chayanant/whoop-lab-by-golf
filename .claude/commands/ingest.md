# /ingest — นำ transcript เข้าคลังความรู้

เป้าหมาย: เปลี่ยน transcript ดิบ (copy จาก NotebookLM) ให้เป็นโน้ตที่กลั่นแล้วใน `vault/`

## Input — แปะ transcript ดิบยาวๆ เข้ามาตรงๆ (อย่าสรุปก่อน)
- ถ้าผู้ใช้วางไฟล์ไว้ใน `inbox/` → อ่านไฟล์นั้น (ถ้ามีหลายไฟล์ ถามว่าจะทำไฟล์ไหน)
- ถ้าผู้ใช้วาง transcript มาในแชทโดยตรง → ใช้อันนั้นได้เลย
- ถ้าไม่มีทั้งคู่ → บอกผู้ใช้ให้วาง transcript ใน `inbox/` ก่อน
- **สำคัญ:** transcript มาจาก Speech-to-Text อาจมีคำผิดเล็กน้อย ให้เข้าใจเนื้อหาโดยไม่ต้องติดคำผิด
  และ **อย่าให้ผู้ใช้สรุปก่อน** — เราต้องการ transcript ดิบเต็มๆ เพื่อรักษาตัวเลข/ชื่อ/รายละเอียด
  (สรุปซ้อนสรุปจะทำให้ atomic notes เสียคุณภาพ)

## ขั้นตอน
1. **Clean** — ตัดบรรทัดว่าง, timestamp, หัวข้อซ้ำ, โฆษณา/sponsor read ที่ไม่เกี่ยวข้อง
2. **ถาม metadata ที่ขาด** — ชื่อคลิป, ผู้พูด/ช่อง, URL (ถ้าผู้ใช้ยังไม่ให้)
3. **สร้าง Source note** ใน `vault/30-Sources/<ชื่อคลิป-kebab>.md` ตาม `vault/templates/source-note.md`
   โดยเรียบเรียงเป็น **ภาษาไทยอ่านง่าย จับประเด็นครบ ไม่ตัดเนื้อหาสำคัญ** ตามโครงนี้:
   - **สรุปภาพรวม** — 2-3 ประโยคว่าคลิปพูดเรื่องอะไร
   - **เนื้อหาตามลำดับ** — หัวข้อหลักแต่ละอัน + ประเด็นสำคัญ + รายละเอียด + ตัวอย่าง (ถ้ามี)
   - **ข้อมูลสำคัญที่ควรจำ** — bullet ประเด็นสำคัญที่สุด (เก็บตัวเลข/protocol เป๊ะ)
   - **ข้อมูลอ้างอิง** — ชื่อบุคคล หนังสือ งานวิจัย ที่ถูกกล่าวถึง
4. **แตก Atomic notes** — ระบุแนวคิดใหม่ในคลิป แล้วสร้าง:
   - `vault/10-Concepts/<ชื่อ>.md` (กลไก/ทฤษฎี) ตาม `concept-note.md`
   - `vault/20-Protocols/<ชื่อ>.md` (วิธีทำจริง) ตาม `protocol-note.md`
   - **ก่อนสร้างใหม่ ให้เช็คว่ามีโน้ตเดิมอยู่แล้วหรือไม่** — ถ้ามี ให้ *เติม/อัปเดต* แทนสร้างซ้ำ
5. **ลิงก์** — ใส่ `[[wikilink]]` เชื่อมโน้ตที่เกี่ยวข้อง และลิงก์กลับไป source เสมอ

## กติกา
- ภาษาไทยเป็นหลัก คงศัพท์เทคนิคอังกฤษ (REM, Adenosine, Circadian Rhythm...)
- อย่าเติมข้อมูลที่ไม่ได้อยู่ใน transcript — ถ้าไม่แน่ใจตั้ง `confidence: medium/low`
- จบด้วยการสรุปว่าสร้าง/อัปเดตโน้ตอะไรบ้าง และแนะนำให้รัน `/wiki-link` ต่อ
- ถามผู้ใช้ก่อนว่าจะให้ลบไฟล์ดิบใน `inbox/` ทิ้งไหมหลังทำเสร็จ
